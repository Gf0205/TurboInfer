# 架构设计笔记

## 高层组件

**Tokenizer 和模型加载器**：

- 加载模型文件
- 准备输入提示词
- 将生成的 token ID 转换回文本

**执行引擎**：

- 运行 Prefill
- 运行 Decode
- 管理 KV Cache
- 协调选定的优化内核

**调度器**：

- 接收传入的请求
- 形成解码批次
- 跟踪进行中和已完成的请求
- 应用公平性和容量策略

**KV Cache 管理器**：

- 分配缓存块
- 跟踪请求所有权
- 完成时释放块
- 记录内存统计

**服务层**：

- 暴露生成 API
- 接收 OpenAI 兼容的请求格式（最终版本）
- 返回生成的文本和时间元数据

当前第一版服务层使用 FastAPI 提供 `/health` 和 `/v1/completions`，详见 [serving-api.md](serving-api.md)。

**基准测试层**：

- 生成工作负载
- 测量延迟和吞吐
- 存储结果表
- 将 TurboInfer 与 vLLM 对比

## 数据流

1. 用户请求进入服务层
2. Tokenizer 将提示词文本转换为输入 ID
3. 调度器将请求放入等待队列
4. 执行引擎运行 Prefill 并创建初始 KV Cache
5. 调度器将活跃的解码步骤批处理
6. 执行引擎生成 token 直到每个请求完成
7. KV Cache 管理器释放内存
8. 服务层返回生成的文本和时间指标

## 设计原则

让每个优化都可测量。每个重大变更应包含基准测试结果和对所针对瓶颈的简要说明。
