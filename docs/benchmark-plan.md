# 基准测试计划

## 核心指标

- **TTFT**：首个 token 生成时间（Time To First Token）
- **TPOT**：每个输出 token 的时间（Time Per Output Token）
- **tokens/s**：每秒生成的 token 数
- **req/s**：每秒完成的请求数
- **P50 延迟**
- **P95 延迟**
- **P99 延迟**
- **峰值 GPU 内存**
- **OOM 前的最大并发请求数**

## 标准测试矩阵

**提示词长度**：

- 128 tokens
- 512 tokens
- 2048 tokens

**输出长度**：

- 32 tokens
- 128 tokens
- 512 tokens

**并发度**：

- 1 个请求
- 4 个请求
- 8 个请求
- 16 个请求
- 32 个请求（硬件允许时）

## 对比规则

每次基准测试应记录：

- 模型名称和版本
- GPU 类型
- CUDA 版本
- PyTorch 版本
- 精度
- 批处理/并发设置
- 提示词长度
- 输出长度
- 随机种子（如适用）

与 vLLM 对比时，使用相同的模型、相同的提示词分布、相同的输出长度和相同的并发度。避免将 TurboInfer 的最佳情况与 vLLM 的最差情况进行比较。

## 硬件解读

**T4**：

- 适合正确性验证和小模型验证
- 用于观察相对趋势
- 不能作为最终性能证据

**3090 / 4090 / A10**：

- 适合迭代优化
- 更适合实际性价比实验

**A100 / H20**：

- 更适合最终基准测试证据
- 更接近生产级推理硬件

## 报告原则

不要只报告加速比。始终包含基线、优化后的数值、工作负载配置和硬件信息。在 infra 面试中，没有工作负载背景的数据是无意义的。
