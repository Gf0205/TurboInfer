# 硬件计划

## 第一阶段：本地机器

**用途**：

- 仓库结构组织
- API 形状设计
- 文档编写
- 尽可能在 CPU 上进行试运行

不要期望这个阶段产生有用的推理性能数据。

## 第二阶段：Google Colab T4

**用途**：

- 验证模型加载是否正常
- 在小模型上验证正确性
- 收集早期的相对基准测试趋势

推荐模型：

- Qwen2.5-0.5B
- Qwen2.5-1.5B
- TinyLlama-1.1B

重要限制：

T4 对开发有用，但不能代表更新的生产推理硬件。请将 T4 数据作为功能验证和早期趋势证据，不要作为最终性能结论。

## 第三阶段：AutoDL 3090 / 4090 / A10

**用途**：

- 迭代性能优化
- 对比优化变体
- 调试内存压力
- 运行中等规模的基准测试表

这些 GPU 更适合实际开发，因为它们通常比 A100 级机器更便宜，同时仍能提供有意义的 GPU 行为。

## 第四阶段：AutoDL A100 / H20

**用途**：

- 最终基准测试运行
- 更强的简历/面试证据
- 更真实地与 vLLM 对比

只有在脚本和配置稳定后再使用此阶段，这样租赁时间花在测量上，而不是调试上。
